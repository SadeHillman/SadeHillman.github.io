CSS Display: inline vs inline-blocks

I chose to research more about the use of "display: inline;" vs "display: inline-block;" because it is one of the things I had difficulty understanding the most! While trying to get the layout of my web page right I noticed when using different element types they all moved differently.

First, to understand what's happening let's look at 2 types of elements:

1. I created the below list using <p> tags. Paragraphs are a "block" element. Block elements take up the whole width of a web page.

2. I created the above links using <a> tags. Links are a "inline" element. Link elements only take up the space of the content.

What does it mean to display: inline;?

Inline elements only taking up the space of its context means we aren't able to add margin, padding, height or width attributes to style. While this isn't very useful when trying to edit your layout it's a great way to prevent any issues with links being bigger or out-of-place when nested in paragraphs.

What does it mean to display: inline-block;?

Inline-block makes it possible for us to take inline elements and display them as inline-level blocks. This means it will hold the inline format but now we can add margins, padding, height and width. This is perfect for displaying multiple links in a button style.

Conclusion

Both inline and inline-block displays have their important roles throughout our web page layouts! One helps keep our page manageable and easy to continue adding CSS throughout while the other allows us to create beautiful design elements.
